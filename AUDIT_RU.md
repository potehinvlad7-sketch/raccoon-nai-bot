# Аудит Raccoon NAI Bot

Дата аудита: 2026-06-28.

## Что сейчас происходит

- Бот хранит пользователей, историю, избранное, последние metadata/payload и админ-конфиг в JSON-файлах внутри `data/`.
- Генерация идёт через `NovelAIClient`: собирается payload, отправляется запрос в NovelAI Image API, zip/png-ответ сохраняется в `data/generated`, затем отправляется пользователю.
- Для обычных пользователей действуют дневной лимит и cooldown; админы обходят эти ограничения.
- Есть админ-панель со статистикой, пользователями, покупками/балансами, рассылкой и отладкой NovelAI payload.

## Исправлено по результатам аудита

- Платные генерации теперь реально учитываются в лимитах: если бесплатный дневной лимит исчерпан, бот расходует `paid_generations_balance`.
- Добавлено поле `paid_generations_used`, чтобы админская статистика не показывала `not implemented`.
- Админская карточка пользователя теперь показывает не только остаток платных генераций, но и фактически использованные платные генерации.

## Найденные недоработки и риски

### Критичные

- Нет тестового набора. Сейчас проверка ограничена компиляцией Python-файлов; регрессии бизнес-логики легко пропустить.
- Лимит и списание генерации происходят перед вызовом NovelAI. Если внешний API упал или вернул ошибку, попытка уже может быть засчитана.
- Хранилище JSON подходит для малого бота, но при росте нагрузки будет узким местом: весь файл читается/пишется на каждое изменение.

### Высокий приоритет

- Очередь генераций глобальная на весь процесс. Один долгий пользовательский запрос блокирует всех остальных.
- `data/generated` не чистится по возрасту или размеру, что может заполнить диск.
- В диагностике API пользователю отдаётся фрагмент ответа NovelAI. Токен очищается из payload, но внешний текст ошибки всё равно лучше показывать только админам.

### Средний приоритет

- Часть настроек доступна только через код или скрытые админ-команды; UX для обычного пользователя неполный.
- `uc_preset_id` хранится отдельно от строкового `uc_preset`, из-за чего возможна рассинхронизация с NovelAI.
- Поведение ArtRaccoon-режима зависит от состояния `artraccoon_base_prompt`; если база пустая, пользовательский prompt становится character prompt, а базовый prompt может уйти пустым.

### Низкий приоритет

- Есть backward-compatible wrapper `nai_client.py`; его стоит оставить только если есть внешние импорты.
- В коде есть смешение русских и английских текстов админки.
- Нет отдельной политики retention для metadata/history/favorites.

## Рекомендации

1. Добавить unit-тесты для квот, payload builder, storage helpers и metadata parser.
2. Перенести списание квоты после успешной генерации либо добавлять компенсацию при ошибке NovelAI.
3. Ввести фоновую очистку `data/generated` по TTL и максимальному объёму.
4. Разделить очередь генераций по пользователям или добавить worker-пул с ограничением параллелизма.
5. Вынести JSON storage в SQLite, если бот используется несколькими активными пользователями.
